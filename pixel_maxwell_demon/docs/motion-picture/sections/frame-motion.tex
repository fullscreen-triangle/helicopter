Frame motion in entropy space fundamentally differs from chronological playback. We establish the mathematical framework for entropy-driven frame traversal and dual-membrane path selection.

\subsubsection{Entropy-Driven Playback Position}

Traditional video defines playback position $p \in [0, 1]$ mapping to chronological frame:

\begin{equation}
\text{Traditional: } p \xrightarrow{\phi} t = p \cdot T \xrightarrow{} I_t
\end{equation}

We instead map playback position to entropy space:

\begin{equation}
\text{Entropic: } p \xrightarrow{\Psi} S_{\text{cum}} = p \cdot S_{\text{max}} \xrightarrow{\Phi^{-1}} \{I_{\text{front}}, I_{\text{back}}\}
\end{equation}

where $S_{\text{max}} = S_{\text{cum}}(T)$ is the final cumulative entropy.

\subsubsection{Forward Scrubbing Dynamics}

When playback position increases ($\Delta p > 0$):

\begin{algorithm}[H]
\caption{Forward Scrubbing in Entropy Space}
\begin{algorithmic}[1]
\STATE \textbf{Input:} Current position $p_{\text{curr}}$, target $p_{\text{next}}$ where $p_{\text{next}} > p_{\text{curr}}$
\STATE \textbf{Output:} Next frame $I_{\text{next}}$
\STATE Compute target entropy: $S_{\text{target}} = p_{\text{next}} \cdot S_{\text{max}}$
\STATE Find frame with entropy closest to target:
\begin{equation}
I_{\text{next}} = \arg\min_{I_k} |S_{\text{cum}}(I_k) - S_{\text{target}}|
\end{equation}
\STATE \textbf{Constraint:} $S_{\text{cum}}(I_{\text{next}}) > S_{\text{cum}}(I_{\text{curr}})$
\STATE Use front face (standard chronological path)
\RETURN $I_{\text{next}}$
\end{algorithmic}
\end{algorithm}

This is straightforward: higher position → higher entropy → later chronological frame.

\subsubsection{Backward Scrubbing: The Revolutionary Mechanism}

When playback position decreases ($\Delta p < 0$), we \textit{cannot} decrease entropy. Instead:

\begin{algorithm}[H]
\caption{Backward Scrubbing with Entropy Conservation}
\begin{algorithmic}[1]
\STATE \textbf{Input:} Current position $p_{\text{curr}}$, target $p_{\text{next}}$ where $p_{\text{next}} < p_{\text{curr}}$
\STATE \textbf{Output:} Next frame $I_{\text{next}}$
\STATE Compute nominal target: $S_{\text{nominal}} = p_{\text{next}} \cdot S_{\text{max}}$
\STATE \textbf{Problem:} $S_{\text{nominal}} < S_{\text{cum}}(I_{\text{curr}})$ (violates monotonicity!)
\STATE \textbf{Solution:} Switch to back face (alternative forward path)
\STATE Find back face frame with entropy increase:
\begin{equation}
I_{\text{next}} = \arg\min_{I_k \in \text{BackFace}} \left\{|S_{\text{cum}}(I_k) - S_{\text{curr}}| : S_{\text{cum}}(I_k) > S_{\text{cum}}(I_{\text{curr}})\right\}
\end{equation}
\STATE Navigate along back face entropy trajectory
\RETURN $I_{\text{next}}$ (from back face, still forward-evolving!)
\end{algorithmic}
\end{algorithm}

\textbf{Key insight}: The back face provides an \textit{alternative entropy path} through the video. When chronological reverse is requested, we instead traverse a different forward path that:
\begin{enumerate}
\item Maintains $S_{\text{cum}}$ increase (thermodynamic consistency)
\item Provides visual content "near" the nominal target (perceptual continuity)
\item Represents valid physical evolution (not time reversal)
\end{enumerate}

\begin{theorem}[Entropy Monotonicity Under Arbitrary Scrubbing]

\begin{equation}
